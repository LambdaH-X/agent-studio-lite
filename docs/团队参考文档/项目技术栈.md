# 项目技术栈分析

## 分类说明
- **重要**：技术栈核心组件，必须一样，否则项目无法合并（例如必须是JAVA17，不能是25或者8）
- **不重要**：辅助工具或库，前面不一样后面可以改（例如maven啥的）

## 后端技术栈

### 重要技术
| 技术 | 版本 | 用途 | 说明 |
|------|------|------|------|
| Java | 17 | 编程语言 | 项目核心开发语言，版本不兼容会导致编译失败 |
| Spring Boot | 3.2.5 | 后端框架 | 项目基础框架，版本差异会影响依赖兼容性 |
| MyBatis | 3.0.3 | ORM框架 | 数据库访问核心框架，版本差异会影响SQL映射（让AI生成它可能改成repository，小心） |
| JWT (jjwt) | 0.11.5 | 认证机制 | 用户身份验证核心（token），版本差异会导致令牌生成/验证失败 |
| Redis | 7.x | 缓存数据库 | 会话和临时数据存储，版本差异会影响命令兼容性 |

### 不重要技术（最好还是一样，因为有些情况是每个都行但是组合在一起就会出问题）
| 技术 | 版本 | 用途 | 说明 |
|------|------|------|------|
| MySQL Connector | 8.0.33 | 数据库驱动 | 仅需在pom.xml中修改版本号即可兼容 |
| SpringDoc OpenAPI | 2.2.0 | API文档 | 仅影响文档生成，不影响核心功能 |
| Lombok | 内置 | 代码生成 | 仅影响开发效率，不影响编译结果 |
| Maven | 4.x | 构建工具 | 版本差异可通过修改配置文件解决 |
| Spring Boot Validation | 内置 | 参数验证 | 辅助功能，版本差异影响较小 |
| Spring Cloud | 2023.0.0 | 微服务支持 | 辅助功能，版本差异影响较小 |

## 前端技术栈（这个是一个人开发的，能和后端接口就行）

### 重要技术
| 技术 | 版本 | 用途 | 说明 |
|------|------|------|------|
| Vue | 3.4.21 | 前端框架 | 项目核心框架，版本不兼容会导致组件语法错误 |
| Element Plus | 2.6.0 | UI框架 | 界面组件核心，版本差异会导致组件API变化 |
| Vue Router | 4.3.0 | 路由管理 | 页面导航核心，版本差异会导致路由配置失效 |
| Pinia | 2.1.7 | 状态管理 | 全局状态管理，版本差异会导致状态API变化 |
| Axios | 1.6.8 | HTTP客户端 | API通信核心，版本差异会影响请求配置 |

### 不重要技术（这个只是推荐，因为比较好看）
| 技术 | 版本 | 用途 | 说明 |
|------|------|------|------|
| Vite | 5.2.0 | 构建工具 | 仅需在package.json中修改版本号即可兼容 |
| @element-plus/icons-vue | 2.3.2 | 图标库 | 仅影响图标显示，不影响核心功能 |
| @vitejs/plugin-vue | 5.0.4 | Vite插件 | 仅影响构建过程，不影响运行结果 |

## 数据库技术栈

### 重要技术
| 技术 | 版本 | 用途 | 说明 |
|------|------|------|------|
| MySQL | 8.0 | 关系型数据库 | 核心数据存储，版本差异会导致SQL语法不兼容 |
| Redis | 7.x | 缓存数据库 | 会话和临时数据存储，版本差异会导致命令不兼容 |
| Docker Compose | 3.8 | 容器编排 | 仅影响部署配置，不影响功能实现（最好还是一样） |

### 不重要技术
| 技术 | 版本 | 用途 | 说明 |
|------|------|------|------|
| phpMyAdmin | 最新 | 数据库管理 | 仅用于开发阶段，不影响最终结果 |
